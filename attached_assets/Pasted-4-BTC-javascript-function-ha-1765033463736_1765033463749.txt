4. Логика начисления награды
При сборе тройки криптошаров (например, BTC):
javascript
function handleBitcoinTriple(userId, countBtcBeads) {
  if (countBtcBeads <= 0) return;
  
  // Обновить дневной счётчик
  if (user.btc_today_date != today) {
    user.btc_today_sats = 0;
    user.btc_today_date = today;
  }
  
  // Рассчитать награду
  const rewardSats = countBtcBeads * btc_per_bead_sats; // например 3 * 5 = 15
  
  // Проверить лимиты
  const canPay = (user.btc_today_sats + rewardSats <= btc_max_per_user_per_day_sats) 
              && (pool_btc_balance_sats >= rewardSats);
  
  if (!canPay) {
    // Отключить BTC-шары для этого юзера
    user.btc_crypto_enabled = false;
    return;
  }
  
  // Начислить награду
  user.btc_balance_sats += rewardSats;
  user.btc_today_sats += rewardSats;
  pool_btc_balance_sats -= rewardSats;
  
  // Сохранить в БД
}

5. Важные правила
Комбо НЕ влияет на криптонаграды: комбо-множитель (1.5) применяется только к виртуальным Beads, а BTC/ETH/USDT начисляются по базовой цене за шар без умножения.
Гарантия честности: если криптошар появился на поле и собрана тройка, награда начисляется всегда (при условии что crypto_enabled = true). Если фонд или лимит не позволяют, криптошары вообще не должны спавниться для этого юзера.
Расчёт в целых единицах: BTC считай в сатоши (int), ETH в wei (bigint), USDT в центах или decimal с 2 знаками после запятой.
