Обнови игру BeadLine: добавь систему дневных лимитов и фондов для криптошаров (BTC, ETH, USDT) с учётом текущих настроек в админке.
Текущие данные из админки:
Обычный шарик: 5 Beads за шар
BTC шарик: 0.00000005 BTC = 5 сатош за шар
ETH шарик: 0.0000001 ETH за шар
USDT шарик: 0.01 USDT за шар
Комбо: множитель 1.5, макс цепочка 10
Вероятность спавна крипто-шаров: 0.08 (8%)
1. Новые поля в базе данных
1.1. Таблица пользователей — добавь поля дневных лимитов:
text
btc_balance_sats         // общий баланс BTC в сатоши (int)
btc_today_sats          // начислено сегодня в сатоши (int)
btc_today_date          // дата последнего начисления BTC


eth_balance_wei         // общий баланс ETH в wei (bigint)
eth_today_wei          // начислено сегодня в wei (bigint)
eth_today_date         // дата последнего начисления ETH


usdt_balance           // общий баланс USDT (decimal)
usdt_today            // начислено сегодня USDT (decimal)
usdt_today_date       // дата последнего начисления USDT

1.2. Таблица настроек/фондов — добавь:
text
// Фонды проекта
pool_btc_balance_sats        // баланс BTC-фонда в сатоши (int), стартовое значение: 100000
pool_eth_balance_wei         // баланс ETH-фонда в wei (bigint), стартовое значение: 1000000000000000 (0.001 ETH)
pool_usdt_balance           // баланс USDT-фонда (decimal), стартовое значение: 100


// Дневные лимиты на пользователя
btc_max_per_user_per_day_sats    // дневной лимит BTC на юзера в сатоши (int), стартовое значение: 300
eth_max_per_user_per_day_wei     // дневной лимит ETH на юзера в wei (bigint), стартовое значение: 3000000000000000 (0.003 ETH)
usdt_max_per_user_per_day        // дневной лимит USDT на юзера (decimal), стартовое значение: 3


// Лимиты криптошаров на игру
btc_max_beads_per_game          // максимум BTC-шаров за игру на юзера (int), стартовое значение: 15
eth_max_beads_per_game          // максимум ETH-шаров за игру на юзера (int), стартовое значение: 15
usdt_max_beads_per_game         // максимум USDT-шаров за игру на юзера (int), стартовое значение: 15

2. Новые поля в админке (вкладка "Экономика")
Добавь редактируемые поля в секцию после "Крипто-шарики":
Секция "Дневные лимиты пользователя":
BTC лимит (сатоши/день): поле ввода btc_max_per_user_per_day_sats
ETH лимит (wei/день): поле ввода eth_max_per_user_per_day_wei
USDT лимит (USDT/день): поле ввода usdt_max_per_user_per_day
Секция "Лимиты криптошаров на игру":
BTC шаров на игру: поле ввода btc_max_beads_per_game
ETH шаров на игру: поле ввода eth_max_beads_per_game
USDT шаров на игру: поле ввода usdt_max_beads_per_game
Секция "Балансы фондов" (read-only для отображения):
BTC фонд (сатоши): отображение pool_btc_balance_sats
ETH фонд (wei): отображение pool_eth_balance_wei
USDT фонд: отображение pool_usdt_balance
3. Логика проверки перед стартом игры
Перед началом каждой игры для конкретного пользователя проверяй флаги btc_crypto_enabled, eth_crypto_enabled, usdt_crypto_enabled:
javascript
// Псевдокод проверки для BTC (аналогично для ETH и USDT)
function checkBtcEnabled(userId) {
  // Обновить дневной счётчик если сменилась дата
  if (user.btc_today_date != today) {
    user.btc_today_sats = 0;
    user.btc_today_date = today;
  }
  
  // Проверить фонд и личный лимит
  const minReward = 3 * btc_per_bead_sats; // минимум за тройку = 15 сатош
  const hasFund = pool_btc_balance_sats >= minReward;
  const hasLimit = user.btc_today_sats < btc_max_per_user_per_day_sats;
  
  return hasFund && hasLimit;
}

Если checkBtcEnabled() == false, то флаг btc_crypto_enabled = false и BTC-шары не спавнятся в этой игре для этого юзера.
